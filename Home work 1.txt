<h1>Base</h1>
<ul>
<li>git config --global user.name blabla ім'я того хто робить цей створив новий commit</li>
<li>git config --global user.email blabal@bla імейл толо хто сворює новий commit</li>
<li>git init - команда для створення архіву з версіями данних</li>
<li>git status - команда яка говорить про стан данних в каталозі</li>
<li>git add - збарігає данні(додати всі файли add .) </li>
<li>git commit - ім'я версії даних і доває його до архіву (commit -m"назва нового коміту"</li>
<li>git log - логи наших і історя commit </li>
<li>git branch - дає можливість працювати/створювати нові гілки</li>
<li>git checkout - дає можливісь дивитись старі версії</li>
<li>git push - відправити нову версію на сервер</li>
<li>git pull - стягнути нову версію з серверу</li>
<li>git merge - Це обеднання двух версій коду в один branch</li>
<li>git clone - витягнути фали з github(копіює до нас окрему версію архіву)</li>
</ul>
<p> .gitignore файл - зкривають файли з якими ми не будемо праювати</p>
<h1>Advanced</h1>
<li>git revert - Видаляє(відкатує) останій commit зі всіми данними, але залишає в логах</li>
